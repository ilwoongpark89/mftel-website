# MFTEL Dashboard — 기능 로드맵

> 2026-02-10 작성. "세계 최고 수준 대시보드"를 위한 기능 우선순위.

## Tier 1 — 필수 (없으면 "좋은 대시보드"에 머무름)

| # | 기능 | 설명 | 상태 |
|---|------|------|------|
| 1 | **Cmd+K 글로벌 검색** | 모든 뷰(논문, 실험, 보고서, 채팅 등) 통합 검색 + 최근 항목 | ⬜ |
| 2 | **알림 센터 + 읽지 않음 뱃지** | 사이드바 뱃지, 알림 패널, 변경사항 자동 감지 | ⬜ |
| 3 | **@멘션** | 채팅/댓글에서 `@이름` → 하이라이트 + 알림 연동 | ⬜ |
| 4 | **활동 피드 (Activity Log)** | "오늘 뭐가 바뀌었나?" 한눈에 파악. 기존 수정 로그 활용 | ⬜ |

## Tier 2 — 뛰어난 대시보드

| # | 기능 | 설명 | 상태 |
|---|------|------|------|
| 5 | **대시보드 Analytics** | 논문 파이프라인, 실험 진행률, 팀별 성과 차트 시각화 | ⬜ |
| 6 | **키보드 단축키** | Cmd+K 검색, 방향키 탐색, Esc 닫기, 숫자키 탭 전환 | ⬜ |
| 7 | **읽음 확인 / 최종 접속 위치** | 채팅 읽음 마커, 새 글 표시 | ⬜ |
| 8 | **리치 텍스트 에디터** | 마크다운/서식(볼드, 링크, 코드블럭) 지원 | ⬜ |

## Tier 3 — 세계 최고 수준

| # | 기능 | 설명 | 상태 |
|---|------|------|------|
| 9 | 반복 작업 / 템플릿 | 주간 보고서, 정기 실험 자동 생성 | ⬜ |
| 10 | PDF/엑셀 내보내기 | 연구 보고서 한 클릭 생성 | ⬜ |
| 11 | 오프라인 + PWA | 네트워크 끊겨도 작동, 모바일 홈화면 앱 | ⬜ |
| 12 | 다크 모드 | 사용 품질 + 상품화 기본 기대치 | ⬜ |
| 13 | 캘린더 뷰 통합 | 일정/휴가/학회 달력 + Google Calendar 연동 | ⬜ |
| 14 | 모바일 최적화 | 반응형 레이아웃, 터치 인터랙션 | ⬜ |

## 구현 순서

```
Phase 1: 검색 + 알림 기반
  1. Cmd+K 글로벌 검색
  2. 알림 센터 + 뱃지
  3. @멘션
  4. 활동 피드

Phase 2: 분석 + 생산성
  5. Analytics 개요 차트
  6. 키보드 단축키
  7. 읽음 확인
  8. 리치 텍스트

Phase 3: 상품화 준비
  9~14. PWA, 다크모드, 내보내기, 캘린더, 모바일, 템플릿
```
